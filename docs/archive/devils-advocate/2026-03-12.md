### 2.2 The Commit Date vs. Filename Date Discrepancy

A specific, previously unexamined issue:

**`analysis-2026-03-11.md`** — Filename states March 11, 2026.

**Commit metadata:**
```
date: 2026-03-10T21:48:39Z (UTC)
```

March 10, 2026 21:48 UTC = **March 11, 2026 03:18 IST**

The commit occurred on **March 10 UTC** but **March 11 IST**. The filename reflects IST. The commit log reflects UTC.

**Issue:** The repository has no declared timezone standard. GitHub displays commit times in UTC. A reader examining the commit history sees this analysis committed on March 10, but the filename says March 11. This creates a subtle but genuine traceability gap.

**Impact:**
- If someone searches commits for "March 11" content, they may miss this file
- The commit history and the filename tell different stories
- The same issue will apply to **this file** (`2026-03-12.md`), which may be committed on March 11 UTC (March 12 IST)

**Recommendation:** Either adopt UTC consistently for all filenames, or add a metadata header in every analysis file declaring the timezone used.
