## Section 2 — Taxonomy Collapse: The Navigation Fiction

**2.1 🔴** The `docs/` directory contains four subdirectories: `academic/`, `devils-advocate/`, `interactions/`, and `people/`. These four directories appear visually equivalent in a directory listing — they share the same level, the same naming convention, and the same implied status. In practice, they are radically asymmetric: `devils-advocate/` contains 33 files; the other three combined contain 3 files. Any user navigating the repository's `docs/` directory encounters a structure that implies four comparable categories of documentation and finds instead one dominant archive and three vestigial stubs.

**2.2 🟡** The three non-analysis subdirectories each contain exactly one file and have not grown since their creation. `docs/academic/` was created February or early March 2026; `docs/interactions/` was created February 24, 2026; `docs/people/colleagues/` was created February 23, 2026. All three have been static for at least 30 days. The directory structure was built to scale and has not scaled. The architecture anticipates content that never arrived.

**2.3 🟡** The `docs/interactions/` directory name implies a collection of documented interaction sessions. It contains exactly one file, documenting one session from February 24, 2026. No sessions since have been documented here. If the intent was to log ongoing AI-assisted interactions for future reference, that intent was expressed once and then abandoned. The directory is now a record of an abandoned practice rather than an active one.

**2.4 🟢** The overall directory taxonomy — root-level files + `docs/` subdirectories — is coherent in the abstract: golden rules and email formatting at root level (operational standards), and organized reference material under `docs/`. But the implementation reveals that the `docs/devils-advocate/` archive has grown to the point where it dominates the taxonomy. The taxonomy was designed for a repository with growing content across all sections; it was not designed for a repository where one section grows without limit and all others are frozen. The architecture has no mechanism to rebalance.
