## Section 4 — New Observations

**4.1 🔴** The `add-task-delegation-golden-rule` branch has existed for 40 days. PR #1 on that branch has been open for 40 days. The branch contains only Golden Rule #1; Golden Rule #2 was added directly to main after the branch was created. If the branch is ever merged now, the merged state of `golden-rules.md` would **overwrite** the main branch version — deleting Golden Rule #2 from the file, since the branch predates Rule #2's creation. A merge of PR #1 would silently delete an operational rule. This has not been identified in any prior analysis.

**4.2 🟡** The file `professional-email-formatting.md` mandates that key points be bolded with HTML `<b>` tags specifically because "markdown may not render in all email clients." This is a technically defensible rationale for email composition. However, the file is read inside GitHub, where Markdown renders and HTML `<b>` tags also render — but are semantically redundant with `**bold**`. The file's reasoning is internally consistent for its *application context* (email clients) but produces unnecessary complexity in its *storage context* (GitHub Markdown renderer). The format choice is correct for emails but odd for the documentation file itself.

**4.3 🟢** `docs/academic/course-codes.md` has a version number (1.0) and a review date. No other file in the repository has a version number or review date. This makes `course-codes.md` the only file in the repository with a built-in staleness signal — and also the only file whose review process is unspecified. A review date without a reviewer or review protocol is a deadline attached to no one.

**4.4 🟡** The repository's `docs/` subdirectory structure (`academic/`, `devils-advocate/`, `interactions/`, `people/`) implies a taxonomy of four peer categories. In practice: `devils-advocate/` contains 33 files (after this commit); the other three combined contain 3 files. The taxonomy is structurally symmetrical but contentually lopsided at a 33:3 ratio. The directory structure advertises a balanced content architecture that has never existed.
